# Values Stress Test

> If a value only holds when it's convenient, it's not a value — it's a preference.

## Protocol
1. Name the value you want to test. Write it as a one-sentence statement: "I value X."
2. Construct the worst-case scenario where holding this value costs something real — social approval, money, comfort, a relationship, a professional opportunity. Be specific. The scenario must feel plausible, not cartoon-extreme.
3. Step into the scenario in writing or imagination. Spend 2–3 minutes fully inhabiting it — feel the pressure, the trade-off, the specific cost of holding the value.
4. Ask: In this scenario, do you still choose this value? If yes, why — write the reason. If not, or if you notice yourself hedging, write exactly where and why the value weakens.
5. Interpret the result:
   - Value holds → it's load-bearing. Note the reason it holds — this is the value's root, more fundamental than its label.
   - Value softens or conditions → classify it as a preference, not a core value. Either remove it from your core values list or specify the conditions under which it applies.

## When to Use
- Before making a major commitment that will require the value under pressure (new role, new creative project, public position)
- When you notice a value you claimed but haven't actually protected — this signals it may be conditional
- As an annual review of your stated values list — to cull the preferences and keep only what's been tested

## Key Insight
> "A value that survives your worst-case scenario is a foundation. Everything else is decoration."
