# One-Click Session Launch

> Most focus attempts abort at the entry point — collapse the start to a single trigger and remove the last moment of friction.

## Protocol
1. Map your current session-start process: list every distinct action between "I decide to work" and "I am actively working on the target output." This typically includes 5-12 steps. Be granular — include unlocking devices, opening applications, navigating to files, adjusting settings, silencing notifications.
2. Automate or pre-configure everything on the list that does not require in-the-moment decision. For digital work: create a workspace shortcut, browser profile, or app shortcut that opens your work environment in the correct state. macOS Shortcuts, Windows Task Scheduler, a browser profile with specific tabs, a script that opens specific applications and files — any of these can collapse multiple steps into one.
3. The physical component: define the single physical action that triggers the launch. Options: pressing the keyboard shortcut, pressing play on a specific playlist, placing a specific object on the desk, pressing a timer button. The action must be specific and small enough to perform with zero deliberation.
4. Sequence the automation so that executing the trigger produces: (a) all digital tools open and positioned correctly, (b) communication channels silenced or closed, (c) the primary document or working file in focus. Test it until one action produces a fully ready environment.
5. Add one micro-ritual at the end of the launch sequence — something physical that completes in 5-10 seconds and is exclusively associated with focus work beginning. Examples: three slow breaths, writing the session's intended output on an index card, saying the intended output aloud. This closes the transition and signals "session active."
6. Keep the launch sequence stable. Changing it frequently degrades both the automation and the conditioned state-entry signal.

## When to Use
- When you regularly decide to do focused work and then find yourself having not started 15-30 minutes later
- When session start involves repeated small decisions about tool state, file location, or environment setup
- Before establishing any new sustained focus practice — build the one-click launch first, before building any other system around it

## Key Insight
> The session rarely fails because you lacked motivation — it fails because the start cost was high enough to let hesitation win.
