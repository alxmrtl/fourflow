# Energy-Mode Tool Swap

> Your cognitive state changes every 90 minutes whether you acknowledge it or not — the tools should change with it.

## Protocol
1. Track your cognitive energy pattern for three days. Every 90 minutes, note your subjective energy level on a 1-5 scale and the quality of your ability to sustain focused thought. Most people have 2-3 high phases per day (often morning, mid-afternoon, and sometimes early evening) with low phases between.
2. Map your identified high and low phases onto your schedule as approximately recurring windows. These will not be exact but provide a working model.
3. Pre-assign tools and task types to each phase based on the task-tool fit matrix. High phases: deep analytical tools, single-document environment, communication closed. Low phases: administrative tools, communication open, review and light processing tasks.
4. Define the swap trigger: a set of physical and digital actions that constitute transitioning between phases. It should take under 90 seconds. Example low-phase entry: save and close primary document, open inbox, open task manager, set a 45-minute timer for the low-phase window.
5. Apply the swap proactively — before energy drops to the point of struggling. The signal to swap is the first appearance of difficulty sustaining focus, not the point of complete depletion. Proactive swaps preserve session quality in both phases; reactive swaps occur after the high phase has already degraded.
6. Return to high-phase tools only after the low-phase window completes and a brief physical reset (movement, water, brief outdoor exposure if possible). Do not attempt to transition back before this.

## When to Use
- When a productive morning regularly transitions into a low-output afternoon with the same tools and tasks in place
- When you notice that some hours of deep work produce significantly more output than others, but you cannot identify what changed
- When managing multiple project types in a day — the swap protocol enables higher-quality work in each mode by matching tools to the available cognitive state

## Key Insight
> The error is not running out of energy — it is continuing to use deep-work tools when the capacity that makes them useful has already cycled down.
