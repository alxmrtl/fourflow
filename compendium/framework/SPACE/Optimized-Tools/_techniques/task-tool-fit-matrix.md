# Task-Tool Fit Matrix

> Using the wrong tool for the cognitive mode is invisible friction — the matrix makes the match explicit before the session starts.

## Protocol
1. Define your four primary cognitive modes with concrete examples of the tasks they contain. (a) Deep/analytical: writing, complex problem-solving, architecture decisions, code design. (b) Creative/generative: brainstorming, drafting, sketching, ideating. (c) Administrative/shallow: email, scheduling, filing, reading summaries, low-stakes decisions. (d) Communication/coordination: meetings, calls, async messaging, feedback exchange.
2. List all tools in your current stack. Assign each tool to the cognitive mode(s) it best serves. Most tools have one primary mode and possibly one secondary mode. Be honest about fit — not about the tool's general capability but about which mode it serves without imposing excess overhead.
3. Build the matrix: a simple two-column table with cognitive mode on the left and the specific assigned tools on the right. Include both digital tools and physical setups (e.g., "notepad and pen" may be the right tool for creative generative work for you; a tablet may be better than a laptop for administrative review).
4. Add one rule per mode: a simple behavioral instruction for transitioning into that mode. Example: "Deep/analytical — close all communication tools, one document open, no browser." This makes the matrix a transition protocol, not just a reference.
5. Print or pin the matrix where it is visible at your workspace. Before beginning any work session, glance at the matrix: am I opening the right tool for this cognitive mode?
6. Review the matrix when either your cognitive mode patterns change (new role, new project phase) or when a new tool is added to the stack. Re-assign as needed.

## When to Use
- At the start of a day where multiple different cognitive modes will be engaged — use the matrix as a session-planning tool, not just an in-session reference
- When you notice that certain tools feel slow or friction-heavy for specific tasks — that friction is diagnostic of a mismatch
- When onboarding a new tool — use the matrix assignment process to clarify which mode the tool is solving for before integrating it

## Key Insight
> The tool that makes you most capable in one mode can make you less capable in another — the matrix is the decision, made once, that covers every session.
