# One-In-One-Out Heuristic

> Before acquiring any new item, remove an existing one from the same category — making exit as automatic as entry, preventing accumulation drift without willpower.

## Mechanism
Accumulation happens asymmetrically: items enter through many small, low-friction decisions (impulse purchases, gifts, freebies, "might need this someday") but rarely leave through equivalent automatic decisions. One-in-one-out makes the exit as automatic as the entry, creating system equilibrium without requiring periodic intervention or willpower. The discipline is in maintaining the rule, not in evaluating each item.

## Protocol

1. Establish your target possession level (via initial Possession Purge or conscious choice of minimalism tier).
2. Apply the rule before any new acquisition: "What leaves from this category to make room?"
3. Identify the specific item to remove *before* acquiring the new one — not after.
4. For performance-focused minimalists, apply a stricter filter: the incoming item must be more aligned with craft advancement than the one it displaces. The question is not "what can I remove?" but "is this new item better than whatever leaves?"
5. Execute the removal on the same day or week as the acquisition — never let the outgoing item linger.

## When to Use
As a permanent default rule applied to any non-consumable acquisition. Most critical for: tools and equipment (where upgrades often result in both versions being kept), clothing, books, and physical media. Apply it before gifting or impulse purchases as well — knowing you must remove something is often sufficient friction to prevent unnecessary acquisitions.

## Key Insight
> "Accumulation is the default state; subtraction requires a system to become the default."
