# Time Block Defense

> A time block without explicit entry and exit conditions is just an intention — the conditions are what make it a defended container.

## Protocol
1. For each time block, define a specific entry condition: a physical or behavioral state you must be in before work begins. Options: five minutes of non-work physical movement, three minutes of review of the session's intended output, completion of a brief workspace reset. The entry condition is not negotiable — if you cannot meet it, you cannot begin the block.
2. Define the single intended output for the block: the specific deliverable that exits the session with you. Not "work on X" — "complete the outline for X" or "draft section 2 of Y." One output.
3. Set the block length based on cognitive mode: deep analytical or creative work maximum 90-120 minutes; shallow processing maximum 60 minutes. Use a timer visible throughout the session.
4. Define the exit condition: what signals the session has ended? Options: timer completes, intended output is produced, a physical signal (standing up, saying "done" aloud). The exit condition triggers regardless of whether the work "feels finished."
5. Schedule an explicit transition buffer of 15-20 minutes after each block: no screens, no switching to new tasks. Walk, drink water, sit quietly. This buffer is part of the block architecture, not optional.
6. Treat the entry and exit conditions as non-negotiable as the calendar event itself. Breaking them degrades the predictability the nervous system uses to prepare for and recover from focused work.

## When to Use
- When time blocking is already part of your schedule but focus depth within the blocks is inconsistent
- When you regularly extend sessions beyond their intended end time and notice diminished returns
- When the warm-up phase consumes a disproportionate fraction of the scheduled block

## Key Insight
> The session does not begin when the clock starts — it begins when the entry condition is met; the defense is in those conditions, not the calendar event.
