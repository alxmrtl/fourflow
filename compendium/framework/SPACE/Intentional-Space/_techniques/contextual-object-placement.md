# Contextual Object Placement

> Objects carry the meaning you have assigned them — place the right ones where your eyes land and the environment does the cueing.

## Protocol
1. For any extended project or focus session, identify the core intent at two levels: the specific output (what you are producing) and the underlying meaning (why it matters). The specific output drives task selection; the underlying meaning sustains motivation during resistance. You need objects priming both.
2. Select one physical object for each level. For the specific output: something that represents the deliverable concretely — a printout of the brief, a physical mockup, an index card with the single output statement written on it. For the underlying meaning: something personal that carries the associated value or purpose — a photo, an artifact, a symbol, a handwritten note.
3. Position both objects in the primary line of sight from your working position — the zone your eyes naturally settle in during pauses or micro-breaks. Not hidden on the desk, not off to the side — in the natural resting position of your gaze.
4. Remove objects not relevant to the current session from this zone. The priming effect of a previous project's artifact competes with the current project's priming if both are in the visual field simultaneously.
5. Update the objects when projects or sessions change. The specific output object should change every session if needed; the underlying meaning object can remain stable across sessions if the same value is engaged.
6. Test the placement by sitting in your working position and noting what your eyes land on during a natural pause. If it is not a contextually relevant object, adjust the placement.

## When to Use
- At the start of any new project phase where maintaining motivational context through resistance is anticipated
- When the connection between daily sessions and larger purpose tends to fade — contextual objects maintain that connection without requiring re-activation through reflection
- When working on long-horizon projects where the original motivation needs structural support to remain accessible over months

## Key Insight
> The environment is always priming something — decide what it's priming before it decides for you.
