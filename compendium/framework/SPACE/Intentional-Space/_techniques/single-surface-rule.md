# Single Surface Rule

> One active surface, one active project — the rest of the world is a different tab you haven't opened yet.

## Protocol
1. At session start, identify the single output you are producing in this session. Not the project — the specific deliverable.
2. Physical surface: have only materials related to that deliverable on the desk. If other projects are physically present, move them to a different surface, a drawer, or a shelf — out of the active visual field.
3. Digital surface: close all applications not required for the current deliverable. Not minimize — close. Minimized applications generate ambient pressure through taskbar presence, notification indicators, and the knowledge of their presence. Close everything except the specific tools for this session.
4. If the work requires reference materials, have exactly one reference document open alongside the primary working document. Not a browser with tabs — one document.
5. When you need to context-switch to a different project (it will happen), complete the switch fully: close the current session's applications, clear the physical surface, open only the materials for the new context. Do not run two contexts simultaneously.
6. Apply this rule to digital device surfaces as well: if a phone is present, it should display a lock screen or be face-down — not a home screen with notification badges.

## When to Use
- Any session where a specific deliverable is the target — not exploratory sessions where scanning multiple contexts is the task itself
- When you notice session productivity is inconsistent and cannot identify a clear reason — ambient surface complexity is often the invisible variable
- When transitioning between project phases where the previous phase's materials are still present in the workspace

## Key Insight
> You cannot fully inhabit one context while another is visible — the eye goes where objects are, and attention follows the eye.
