# Live Output Check

> Drift compounds — catching it mid-session costs a minute; catching it at the end costs hours.

## Protocol
1. Before beginning a work session, write the intended output as a single concrete statement. Not "work on the article" but "produce a complete draft of section 3, making the specific argument that X causes Y through mechanism Z." Keep it visible.
2. Set a check interval: for sessions up to 60 minutes, one check at the midpoint. For sessions over 60 minutes, one check every 30-40 minutes.
3. At each check point, stop working completely. Read the intended output statement you wrote at the start.
4. Ask: "Is what I am currently producing still tracking this intent?" Answer with yes, partial, or no — not with explanation.
5. If yes: continue. If partial: identify the specific point where execution diverged and decide in 60 seconds whether to redirect or update the intent (sometimes the divergence is an improvement). If no: stop and trace back to the last point where execution was on intent, then restart from there.
6. Do not skip the check even when the session feels productive. Feeling productive while drifting is the primary failure mode this protocol addresses.

## When to Use
- Any session longer than 30 minutes where there is a specific intended output, not just a general topic
- When working under conditions that reduce meta-cognitive capacity: fatigue, high emotional load, unfamiliar material
- After a pattern of sessions that produced output but not the specific output needed

## Key Insight
> The question is not whether you are working hard — it is whether what you are producing is still what you meant to produce.
