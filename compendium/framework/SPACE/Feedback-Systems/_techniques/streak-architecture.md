# Streak Architecture

> A visible chain converts each session into the act of not breaking it — loss aversion becomes your consistency engine.

## Protocol
1. Choose one specific behavior for the streak. It must be completable in a defined daily window and have a binary outcome — done or not done. "Wrote something" is too ambiguous; "produced at least one complete paragraph" is trackable.
2. Define the minimum viable rep — the smallest version of the behavior that still counts as a full day. This is not the target; it is the floor. On hard days, the floor is what keeps the chain alive. Example: "200 words minimum, target 500."
3. Set up a physical display (paper calendar on the wall, marked with an X per day) or a digital display (habit tracking app, spreadsheet, or whiteboard photo updated daily). The display must be in your primary work environment — visible without specifically looking for it.
4. After each completed rep, update the display immediately. The update is part of the habit, not an optional administrative step.
5. Write the repair protocol now, before you need it: "If I miss one day: I do double the minimum viable rep the next day and the chain continues." One missed day = chain continues with a repair rep. Two consecutive missed days = the streak counter resets, but the habit does not stop. Distinguish between the streak (a display mechanism) and the habit (the actual target behavior).
6. Review the streak count weekly and set a rolling target: "I will reach X days by this date." This creates a second forward-looking goal alongside the backward-looking chain, which maintains approach motivation when the chain is already long and the loss-aversion effect diminishes.

## When to Use
- When establishing a new practice that requires daily consistency before results become intrinsically motivating
- When existing motivation is primarily outcome-based (far future) and needs a nearer-term visible feedback loop
- When you have a pattern of starting practices and discontinuing them after the first disruption

## Key Insight
> Design the break before it happens — the streak is a motivational tool, and a repair protocol is part of its design, not a concession to failure.
