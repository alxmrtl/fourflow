# Peer Calibration Check

> Your internal sense of "good enough" drifts without an external reference point — calibrate it deliberately.

## Protocol
1. Identify one specific skill domain you want to calibrate (not "overall performance" — too diffuse). Example: written argument clarity, code architecture decisions, presentation pacing.
2. Select a peer or comparator who you believe is one level above you in that specific domain. This can be a person, a body of published work, or a defined benchmark standard. The comparator must be directly observable — you need to see their actual output, not just their reputation.
3. Obtain a specific comparable sample of their work in that domain. A single piece is sufficient; the goal is concrete observation, not statistical sampling.
4. Complete a sample of your own work in the same domain, using the same parameters (same task type, same time constraint if possible).
5. Do a side-by-side comparison. Write down three specific observable differences — not "theirs is better" but specific structural, technical, or craft differences you can name.
6. From those three differences, extract one thing you are currently underweighting in your own work. This is the calibration output.
7. Run this protocol monthly at minimum for any domain you consider a core competency.

## When to Use
- After a period of working in isolation — when external feedback has been sparse and self-assessment has been the primary input
- When receiving contradictory feedback (some people say it's great, others say it needs work) and you cannot tell which is accurate
- Before setting goals or defining a new performance target in a skill domain

## Key Insight
> The internal scale you use to judge your work is only as accurate as the last time you compared it to something real.
