# Role Values Map

> When daily tasks feel disconnected from who you are, make the connection explicit — it usually exists but is invisible.

## Protocol
1. List your 5 core values — not aspirational values, but the ones your behavior already reflects (use behavioral data if needed).
2. List your 10 most frequent role activities (what you actually spend time on).
3. Draw a map: for each activity, draw lines to the values it serves. An activity can connect to multiple values; a value can receive multiple connections. Some activities may have no direct line — note those.
4. For activities with no direct value connection: ask "could I do this activity in a way that expresses [core value]?" Often the connection exists at the level of how the activity is done, not just what is done.
5. For values with no or few connections: identify which current activities could be reframed or which new activities could be added to close the gap.
6. Keep the map visible. Update it quarterly or when the role significantly changes.

## When to Use
- When a role that was once motivating has started to feel hollow or perfunctory
- When considering a significant role change — to map what values the current role serves before assuming the next one will
- During onboarding to a new role or project — establishing the values connection early prevents the drift into hollow performance

## Key Insight
> "Meaning isn't in the task — it's in the visible line between the task and who you are."
