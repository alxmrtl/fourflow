# Role Boundary Declaration

> Unclear role edges don't just create awkwardness — they split attention between doing the work and monitoring for violations.

## Protocol
1. Write the declaration header: "I own:" — list every responsibility, decision, output, and domain that is unambiguously yours.
2. Write: "I do not own:" — list the adjacent responsibilities, decisions, and domains that might seem related but belong to others or to no one.
3. Write: "I am a contributor to, but not the owner of:" — list collaborative or shared responsibilities and name who the actual owner is.
4. Review for gaps: anything that appears in none of the three lists is a genuine ambiguity that needs to be resolved with the relevant party.
5. Share the declaration with anyone whose work interfaces with yours — invite corrections. Update as roles evolve.
6. At the start of any session where role-boundary anxiety has been present, read the "I own" list. It reinstates the clear mandate.

## When to Use
- When beginning a new role, project, or collaboration — establish boundaries before ambiguity accumulates
- When anxiety during work sessions has a "should I be doing this?" quality that doesn't resolve through reassurance
- After a period of scope creep where you've been doing things that arguably aren't yours — to reset clarity

## Key Insight
> "You cannot flow in a territory you're not sure is yours."
