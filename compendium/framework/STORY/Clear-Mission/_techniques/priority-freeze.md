# Priority Freeze

> Once the top priority is set, it is not available for re-evaluation until the session ends.

## Mechanism
Priority drift — the pattern of deciding what the most important thing is, then revising that decision mid-session when something more urgent or more interesting presents itself — is one of the primary mechanisms by which sessions lose coherence. Each revision is individually justifiable: new information arrived, something more pressing emerged, the original priority was wrong. But cumulatively, a session with three priority revisions produces output that is a third as deep as one with none.

The psychological mechanism is hyperbolic discounting: the brain systematically overweights present rewards relative to future ones. A task that presents itself now feels more urgent than the task you chose before the session started, even when the pre-session choice was made with better information and clearer judgment. The priority set before the session was made by a more deliberate mind; the priority revision made during the session is made by a more distracted, more reactive one.

The freeze operates as a commitment device — a mechanism that binds future choices by raising the cost of deviation. Behavioral economics research (Ariely, Thaler) demonstrates that commitment devices are effective precisely because they remove the option to re-evaluate in the moment. The freeze doesn't claim that the original priority was always right; it claims that the cost of mid-session revision almost always exceeds the cost of finishing with an imperfect priority.

## Protocol
1. Set the session priority before starting work (use the One Thing Card or equivalent).
2. State the freeze explicitly: "This is the priority for this session. It is not available for change until the session ends or I invoke the override protocol."
3. Define the override protocol in advance — what constitutes a legitimate exception (e.g., a genuine emergency, not "I thought of something more interesting"). Write it down. It should be narrow.
4. When a priority revision impulse arrives: note it in a capture system (don't evaluate it), and return to the frozen priority.
5. At session end: review any captured revision impulses. Were they genuine improvements or reactive drift? This calibrates future sessions.

## When to Use
- Any session where you have set an intention before but frequently ended up working on something different
- High-distraction environments where adjacent tasks are constantly visible and competing
- When a project requires consistent depth over multiple sessions — protecting each session's priority is how that depth accumulates

## Key Insight
> "The mind that sets priorities before the session is smarter than the mind that revises them during it."
