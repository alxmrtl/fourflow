# Scope Lock

> What you decide is out of scope before the session starts is unavailable for re-evaluation during it.

## Mechanism
Scope creep — the expansion of what a session is trying to accomplish through the addition of adjacent tasks during work — is one of the most common mechanisms by which focused sessions degrade into scattered ones. The creep is not random; it follows a predictable pattern. During work on task A, task B presents itself as relevant or urgent. Without a pre-existing decision about B's status, the brain engages in an in-the-moment evaluation: should I address this now? This evaluation consumes decision-making resources and, frequently, produces a "yes" that fractures focus.

The pre-declaration of out-of-scope items neutralizes this pattern by converting live decisions into pre-made ones. Decision fatigue research (Baumeister, Vohs) demonstrates that the quality of decisions degrades over a session as executive resources deplete. Decisions made before the session — when those resources are fresh — are higher quality and eliminate the in-session cost entirely. An item declared out of scope before work begins is not evaluated when it appears; it is simply routed to a capture system and the session continues.

There is a secondary mechanism: pre-declaration makes the scope of a session more legible to the prefrontal cortex. A bounded mission is a clearer signal than an unbounded one. The brain can commit more fully to a task with defined edges than to a task that might expand in any direction.

## Protocol
1. Before starting any work session, write the session's mission in one sentence.
2. Write a scope lock list: "Out of scope for this session:" — list at least 3 categories of things that might present themselves as relevant but are not part of this session's mission. (e.g., "Replying to any messages," "Updating the design," "Researching new approaches to X.")
3. For each out-of-scope item: confirm there is a capture system where it can go if it appears (a running list, inbox, etc.) — so it won't be lost.
4. Post the session mission and scope lock where you can see them during the session.
5. When an out-of-scope item appears: capture it (don't evaluate it), return to the session mission. No re-evaluation.

## When to Use
- Before any focused work session, especially those involving creative or complex problem-solving
- When previous sessions have consistently expanded beyond their original scope and left the primary work incomplete
- On days with many adjacent tasks competing for attention — explicitly choosing what is NOT getting done is as important as choosing what is

## Key Insight
> "Every in-session decision about scope is attention stolen from the work. Pre-decide and those thefts stop."
