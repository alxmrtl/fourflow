# Scope Lock

> What you decide is out of scope before the session starts is unavailable for re-evaluation during it.

## Protocol
1. Before starting any work session, write the session's mission in one sentence.
2. Write a scope lock list: "Out of scope for this session:" — list at least 3 categories of things that might present themselves as relevant but are not part of this session's mission. (e.g., "Replying to any messages," "Updating the design," "Researching new approaches to X.")
3. For each out-of-scope item: confirm there is a capture system where it can go if it appears (a running list, inbox, etc.) — so it won't be lost.
4. Post the session mission and scope lock where you can see them during the session.
5. When an out-of-scope item appears: capture it (don't evaluate it), return to the session mission. No re-evaluation.

## When to Use
- Before any focused work session, especially those involving creative or complex problem-solving
- When previous sessions have consistently expanded beyond their original scope and left the primary work incomplete
- On days with many adjacent tasks competing for attention — explicitly choosing what is NOT getting done is as important as choosing what is

## Key Insight
> "Every in-session decision about scope is attention stolen from the work. Pre-decide and those thefts stop."
