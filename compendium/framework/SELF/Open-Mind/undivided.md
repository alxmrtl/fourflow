# Undivided
*The Concentrate quality of Open Mind*

## What it is

When this quality is present, the work gets all of you. No background threads are running, no other tasks are still active in the mind's maintenance queue. You're not working and simultaneously monitoring something else. When absent, flow can't build — not because the task isn't engaging, but because the available attention has already been divided before you begin. It's a quiet split. Often unnoticed until you realize you've been in the session for an hour and it never deepened.

## Science

Sophie Leroy's 2009 research on attention residue demonstrated that people who switched to a new task before fully disengaging from a previous one showed measurably impaired performance on the new task — not because they were consciously thinking about the old task, but because the brain's background processing continued tracking it. The prefrontal cortex keeps an active representation of incomplete goals as memory protection: it will not allow unfinished work to fall out of working memory until the goal is resolved or explicitly parked.

This creates a binding problem for flow: working memory has a capacity of roughly 4±1 items (Cowan, 2001), and each open background thread occupies a slot. A person carrying three active maintenance tasks into a writing session is working at meaningfully reduced capacity — less per-moment perception, slower processing, shallower absorption. The Zeigarnik effect (incomplete tasks stay active in the mind) means this isn't willpower-solvable; it requires explicit structural closure.

The concentrate path is procedural: before switching tasks, close the loop explicitly. Not just stopping, but noting what's done, writing the next step on the prior task, and verbally marking it as parked. The explicit closure satisfies the prefrontal protection mechanism and releases the slot.

## Techniques

- [[open-loop-closure]] — deliberate end-of-task sequence that parks unfinished threads before switching
- [[single-tab-mind]] — constrains active work to one task at a time, preventing residue accumulation
- [[transition-ritual]] — brief structured gap between tasks that allows residue to dissipate before re-engagement

## Recall

**Hook:** Every task you leave incomplete without explicitly parking it is still running in the background — consuming attention you need for the present task.
**Mechanism:** Incomplete tasks generate active representation loops (Zeigarnik effect); eliminating residue closes or explicitly parks these loops before switching, freeing the full bandwidth for absorption.
**Anchor:** Before switching: (1) note what's done, (2) write the next step on the prior task, (3) say "that's parked" — then transition. The explicit closure is the mechanism.
