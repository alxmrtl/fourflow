# Attention Residue Elimination

> Prevents task-switching drag.

## Why It Enables Flow

When you switch tasks without fully closing the previous one, a portion of attention remains stuck on the prior task — still processing it, worrying about what was left incomplete, replaying unresolved threads. This is attention residue: the cognitive drag that prevents full absorption in the current task. Incomplete tasks generate Zeigarnik-effect loops (the mind tends to keep active the representation of unfinished work until it is done or explicitly parked). Flow requires 100% of available attention on the present task; any residue is a leak in that supply.

## Techniques

- [[open-loop-closure]] — deliberate end-of-task sequence that parks unfinished threads before switching
- [[single-tab-mind]] — constrains active work to one task at a time, preventing residue accumulation
- [[transition-ritual]] — brief structured gap between tasks that allows residue to dissipate before re-engagement

## Recall

**Hook:** Every task you leave incomplete without explicitly parking it is still running in the background — consuming attention you need for the present task.
**Mechanism:** Incomplete tasks generate active representation loops (Zeigarnik effect); attention residue elimination closes or explicitly parks these loops before switching, freeing the full bandwidth for absorption.
**Anchor:** Before switching: (1) note what's done, (2) write the next step on the prior task, (3) say "that's parked" — then transition. The explicit closure is the mechanism.
