# Attention Residue Elimination

> Prevent task-switching drag.

## Why It Enables Flow

When you switch tasks without fully closing the previous one, a portion of attention remains stuck on the prior task — still processing it, worrying about what was left incomplete, replaying unresolved threads. This is attention residue: the cognitive drag that prevents full absorption in the current task. Incomplete tasks generate Zeigarnik-effect loops (the mind tends to keep active the representation of unfinished work until it is done or explicitly parked). Flow requires 100% of available attention on the present task; any residue is a leak in that supply.

The neuroscience is precise. Sophie Leroy's 2009 research demonstrated that people who switched to a new task before fully disengaging from a previous one showed measurably impaired performance on the new task — not because they were consciously thinking about the old task, but because the brain's background processing continued tracking it. The prefrontal cortex, which manages goal maintenance, keeps an active representation of incomplete goals as a form of memory protection — preventing them from falling out of working memory until they can be resolved. This is adaptive in low-load environments but becomes costly in deep work, where working memory (Cowan, 2001: 4±1 items) is already near capacity. Each residue thread consumes a slot that would otherwise go to the focal task.

In aperture terms, attention residue narrows the aperture — each background thread running in parallel reduces the available bandwidth per present moment. A person carrying three open loops into a writing session is not working at full aperture; they are perceiving less of the current task per moment than someone who arrived with no active maintenance load. Eliminating residue is not a productivity trick — it is an aperture-clearing operation. The aperture that was partially consumed by background threads becomes fully available for the present signal.

## Recall

**Hook:** Every task you leave incomplete without explicitly parking it is still running in the background — consuming attention you need for the present task.
**Mechanism:** Incomplete tasks generate active representation loops (Zeigarnik effect); attention residue elimination closes or explicitly parks these loops before switching, freeing the full bandwidth for absorption.
**Anchor:** Before switching: (1) note what's done, (2) write the next step on the prior task, (3) say "that's parked" — then transition. The explicit closure is the mechanism.
