# Overchoice Elimination

Make every meta-decision before the session starts — what to work on, in what order, with which tool, for how long — so no decision intrudes during the work itself.

## Protocol

1. Before any focused session, answer in writing: What exactly am I working on? In what order? With what tool? For how long?
2. Limit to one tool per session type. One method. One problem. Deliberately narrow the options available during the session.
3. Batch all project-level decisions (priority, approach, scope) into a single weekly planning block rather than making them reactively throughout the week.
4. During the session, treat all meta-decisions as already made. If a new meta-decision arises, write it down for the planning block and return immediately to the work.

*If choices still feel live during the session, step 1 wasn't specific enough. Add more specificity before the next session.*

## When to Use

- When sessions start well but get interrupted by "what should I do next?" moments
- When you're choosing between multiple approaches or tools during execution
- When decision fatigue is accumulating across the day

## Key Insight

> "Every decision made during flow is a withdrawal from the finite capacity needed for the work itself."

## Recall

**Hook:** Every decision made during flow is a withdrawal from the finite capacity needed for the work itself.
**Mechanism:** Decision fatigue degrades choice quality and consumes the prefrontal bandwidth needed for deep work; pre-made constraints and defaults eliminate decisions at their source, preserving the capacity for the task.
**Anchor:** Before starting any focused session, answer: What exactly am I working on? In what order? With what tool? For how long? Make every meta-decision before the work begins — zero decisions during.
