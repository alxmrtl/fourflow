# Compression Over Expansion

After engaging with any complex idea, distill it into the minimum form that holds the essential meaning — so it can be held and used in working memory without effort.

## Protocol

1. After writing or thinking through any complex idea, ask: "What is the one-sentence version?" Write it.
2. Remove every word from that sentence that doesn't change the meaning. Stop when removing one more word would lose something essential.
3. For domain knowledge: find the phrase, metaphor, or symbol that most immediately activates the full concept. Use that during active work rather than the expanded explanation.
4. Apply the same principle to goals and projects: what is the single essential thing? Name it. Keep only that active during execution sessions.
5. When knowledge acquisition starts generating more complexity than clarity, stop acquiring and start compressing — find the enough point, then act.

*Review compressed versions weekly. Some will need updating as understanding deepens; some will have become unnecessary.*

## When to Use

- After a research or learning session, before moving to execution
- When working memory feels crowded with concepts that slow thinking
- When domain knowledge has expanded to the point where retrieving it consumes noticeable energy

## Key Insight

> "Working memory holds a fixed number of chunks, not a fixed amount of information — compression makes each chunk carry more."

## Recall

**Hook:** Working memory holds a fixed number of chunks, not a fixed amount of information — compression makes each chunk carry more.
**Mechanism:** Encoding more meaning into fewer units (phrases, metaphors, symbols) frees the working memory slots that would otherwise be consumed by expanded knowledge, making more bandwidth available for active problem-solving.
**Anchor:** After writing or thinking through any complex idea, ask: "What is the one-sentence version?" Compress until losing that sentence would lose something essential. Stop there.
