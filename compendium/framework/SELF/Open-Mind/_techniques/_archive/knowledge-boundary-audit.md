# Knowledge Boundary Audit

> Identifies the point where knowledge acquisition stops adding usable capability and starts adding load — the signal to compress and act.

## Mechanism

Working memory compresses integrated knowledge into chunks, but only knowledge that has been applied gets fully integrated. Unacted-upon knowledge stays expanded: it occupies space without becoming usable, and no amount of additional research will compress it. The boundary of practical utility is the point where you're adding volume rather than capability. Beyond that boundary, more learning doesn't reduce uncertainty — it just delays the feedback that only action can provide.

## Protocol

1. Name the decision or action the knowledge is meant to support. If you can't name one, stop acquiring — there's no compression target.
2. Review what you've gathered: "Can I make an acceptably useful call with this?" If yes → stop acquiring, start applying.
3. If no → identify the specific gap (not a category, a precise question), fill only that, return to step 2.
4. After acting, notice what the action taught you that research didn't. That gap is always smaller than anticipated before acting.

## When to Use

When a research or learning phase runs longer than expected. When you're preparing to do something but keep finding more to read. As a pre-session check before any planning or intake period — ask "what action does this knowledge serve?" before opening the first tab.

## Key Insight

> You don't know what you don't understand until you try to use it — acting is the most efficient way to discover the actual knowledge gaps.
