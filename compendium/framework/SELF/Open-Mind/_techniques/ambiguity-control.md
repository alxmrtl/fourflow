# Ambiguity Control

Before any significant work session, define what you're doing, why, and what done looks like — so the mind can stop asking fundamental questions and give itself fully to the work.

## Protocol

1. Complete this sentence before starting: "I'm working on [specific thing] until [specific condition], and I'll know I'm done when [clear signal]."
2. If you can't complete any part of the sentence, clarify that part first — don't start with open questions still active.
3. Distinguish productive uncertainty (not yet knowing how to solve the problem) from paralyzing ambiguity (not knowing whether the problem is right). Address only the latter before starting.
4. Find the minimum viable definition: the smallest amount of structure that lets work begin — enough to start, not so much that the definition itself becomes the work.

*Review at session midpoint if the fundamental questions return. Clarify again before continuing.*

## When to Use

- Before any focused session that involves ambiguous or complex creative work
- When you notice the mind asking "is this the right thing to work on?" during execution (that's paralyzing ambiguity, not productive uncertainty)
- At the start of a new project before scope is clear enough to allow genuine absorption

## Key Insight

> "The mind cannot fully enter a task when it is simultaneously asking whether the task is right."

## Recall

**Hook:** The mind cannot fully enter a task when it is simultaneously asking whether the task is right.
**Mechanism:** Bounded uncertainty (defined goals, known scope) creates the structure within which creative exploration becomes possible; unbounded ambiguity produces the anxiety that closes the mind instead.
**Anchor:** Before starting: complete this sentence — "I'm working on [specific thing] until [specific condition], and I'll know I'm done when [clear signal]." If you can't complete it, clarify before you begin.
