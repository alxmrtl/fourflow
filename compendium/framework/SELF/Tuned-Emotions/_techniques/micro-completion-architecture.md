# Micro-Completion Architecture

> Decompose any project or session into the smallest meaningful units of completion — each one a "win" with a clear before/after — so the dopamine loop closes constantly rather than once at the end.

## Mechanism
Without sub-goals, a long session feels like one continuous effort event — the dopamine loop only closes once at the end. With sub-goals, the loop closes 5, 10, 20 times during the session. Each closure replenishes motivation for the next segment. The session builds momentum instead of depleting it. Csikszentmihalyi identified that flow requires clear sub-goals — not just a terminal outcome, but a sequence of achievable steps that each generate a completion signal. This is effort-reward coupling built into task structure.

## Protocol

1. Before starting any project or session, decompose the work into the smallest meaningful units of completion.
2. A "win" is defined by a clear before/after state: before this sub-goal, X was incomplete; after, X is done.
   - "Write the opening paragraph" (not "work on the opening")
   - "Fix the failing test in module 3" (not "debug the code")
   - "Finish reading chapter 4 and summarize the key point" (not "read for 45 minutes")
3. Work through wins sequentially. Do not move to the next sub-goal while the current one is unfinished.
4. **Acknowledge each completion explicitly** — note it, mark it, say it internally. The acknowledgment is the reward receipt that closes the coupling loop. This step is not optional: silent completion provides weaker dopamine signal than acknowledged completion.

## When to Use
For any session longer than 30 minutes, and for any project where motivation tends to flag mid-session. Particularly useful in the early phase of any project, when natural momentum hasn't built yet. Also effective for "energy debt" sessions — when you're working below your normal capacity and need the session to generate its own forward momentum.

## Key Insight
> "The brain doesn't reward effort — it rewards completion. Structure work so completion happens constantly."
