# Self-Driving Car Iteration Model

> Reframe each attempt as one data point in an iteration sequence — not a test of adequacy, but a system update that brings the output closer to its destination.

## Mechanism
The self-driving car reached its capability through an extraordinary volume of failure — millions of errors processed and integrated before it could reliably navigate a road. The model is useful precisely because it separates iteration from identity entirely. The system is not embarrassed by its errors; it simply updates. Each error refines the parameters. The accumulation of refinements produces capability that no amount of pre-planning could generate. The capability emerges *from* the iteration, not before it.

## Protocol

1. Before attempting something where failure seems likely, state the reframe explicitly: "This attempt is not a test of adequacy. It is one data point in an iteration sequence."
2. During the attempt, if failure occurs, ask: "What parameter updated?" (What did this error tell the system?) Not: "What does this failure say about me?"
3. Apply the update. Run the next iteration from the updated parameters.
4. Track iterations, not results. A person with 50 iterations is ahead of a person with 5 — regardless of the success rate on those iterations — because the parameters are more refined.

## When to Use
When feeling resistance to attempting something because failure seems likely. When a task feels too risky to try. When the gap between "where I am" and "where the task requires" seems embarrassingly large. The model reframes the gap as iteration distance, not character distance.

## Key Insight
> "The self-driving car made billions of mistakes before reaching a meaningful destination. You're not behind — you're mid-iteration."
